SPRING

Instalacion

Spring es un framework para Java, es decir un entorno de trabajo, para facilitar el desarrollo de aplicaciones; es la alternativa de aplicaciones para JEE (Java enterprice).
Dentro de sus ventajas esta la inyección de dependencias, favoreciendo el loose coupling(hacer modificaciones a múltiples clases), además el desarrollo sencillo con POJOs(Plain old Java Objects), también minimiza el código repetitivo, simplifica el acceso a datos, y la programación orientadas a Aspectos. tiene como requisitos: tener el JDK, JRE versión >=8, servidor java(tom cat), IDE(entorno de desarrollo integrado)JAVA. El IDE utilizado debe ser Eclipse EE (Eclipse enterprice edition), y tenerinstalado TomCat como servidor. Procedemos entonces a descargar las Sprin jar files para posteriormenteimportarlas a eclipse

Se descarga la última versión de los spring files que están hasta el final de la hoja;

Luego hacemos click en el primer vinculo, con el que se descarga un archivo (spring-framework-5.2.9.RELEASE) .zip, que posteriormente vamos a extraer

Integrar libreria

Nos colocamos en la vista de Java (window/ perspective/ open perspective) iniciamos un nuevo proyecto: File / new / new Java project, en la ventana que se abre dejamos seleccionado el boton de "use a project specific JRE" y damos en Finish

Ahora el proyecto creado aparece en el listado de carpetas, en el costado izquierdo de la interfaz, damos clic derecho sobre este: build path / configure Build path.

En la ventana que se abre, vamos a la pestaña Order and Export, Colocamos de primero: la carpeta SRC, utilizando los el Boton lateral

Ahora, vamos a la pestaña Libraries, damos un click sobre Modulepath (tiene dos flechitas) A lo cual se habilitan el boton de Add JARs...

Damos click en el Boton Add External JARs..., A lo cual se abre una nueva ventana, donde buscamos la ruta de la carpeta en donde se encuentra el Spring JAR filesque descargamos, en la carpeta libs; solamente seleccionamos los archivos binarios, es decir que no incluimos los -javadoc & -source

Despues de abrir los archivos seleccionados, damos en Apply y despues en Apply and Close

Con lo cual se nos crea un archivo nuevo llamado: JRE System Library

CRUD-mySQL-Spring

Creamos el Initializr, como un proyecto de Maven, en Java, packaging: .JAR incluyendo las siguientes librerias: spring web, Thymeleaf, Srping Data JPA, MySQL Driver
Ademas creamos, descomprimimos e importamos el proyecto a eclipse como Maven.

procedemos a configurar las propiedades de la aplicacion, en la ruta: src/main/resources/templates/application.properties donde el archivo se encuentra vacio. Colocamos la siguiente configuracion para Mysql:

spring.datasource.url=jdbc:mysql://localhost:3306/crud_spring?serverTimezone=UTC
spring.datasource.username=root
spring.datasource.password=
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.jpa.hibernate.ddl-auto=create

Dentro del paquete principal (src/main/java) generamos cinco paquetes para el proyecto: (clic-der * src/main/java* / new /Package) colocando en el nombre: punto + palabra identificadora, generando los paquetes: controler, interfaces, modelo, service & interfaceService

En el paquete .modelo creamos una clase a la que llamamos persona, con tres variables correspondientes a los campos de la tabla de la BBDD

luego con click derecho sbre el codigo de la clase persona, (clic-der / Source /Generate contructor using fields..) y creamos el constructor (clic-der / Source /Generate Getters and Setters) y creamos getters y setters Ademas creamos un constructor sin parametros

sobre la declaracion de la clase, colocamos las anotaciones: (importando las clases, 0cuando aparezca el error); en donde la anotacion Table contiene el atributo name, indicando el mismo nombre de la clase:
@Entity
@Table(name="persona")
luego sobre el parametro correpondiente al "id" de la tabla, en la clase, colo camos las anotaciones:
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)

Ahora en el Package interfaces(com.example.demo.interfaces), creamos una Interfaz, llamada Ipersona, y le heredamos de la clase CrudRepository*Object, Object* Colocando como ..extends CrudRepository*Persona, Integer*{ ademas, a la clase, le agregamos la anotacion: @Repository

Ahora en el paquete interfaceService creamos otra interfaz, a la que llamaremos: IpersonaService, y dentro de ella creamos los metodos del CRUD

public List*Persona*listar();
public Optional*Persona*listarId(int id);
public int save(Persona p);
public void delete(int id);

Teniendo presente se hagan las respectivas importaciones:
import java.util.List;
import java.util.Optional;
import com.example.demo.modelo.Persona

En el Package service (com.example.demo.service), creamos una clase a la que llamamos PersonaService, y le implementamos la interfaz creada anteriorment con sus respectivos metodos. ademas, a la clase, le agregamos la anotacion:
@Service

Ademas creamos una variable de Ipersona llamada data, a la cual le colocamos la anotacion
@autowired
Cuidando se haga las respectivas importaciones ahora en esa clase; dentro del metodo, List<>, colocamos la variable data con el metodo .findAll() , este metodo proviene de -CrudRepository, el cual se debe castear con (List*Persona*)

Ahora en el package controlador(com.example.demo.controlador), creamos una clase llamada Controlador, colocandole las anotaciones:
@Controller
@RequestMapping
ademas le creamos un metodo llamado listar con parametro Model y return "index", refiriendoce al .html, al que le colocamos la anotacion, agregando la ruta ("/listar"):
@GetMapping("/listar")
Tambien creamos una variable de IpersonaService, llamada service y le colocamos la anotacion:
@Autowired
A esta variable le agregamos el metodo listar: service.listar(); asignandola dentro de una variable de tipo List.
Ahora a la variable model le agregamos el medodo addAttribute() incluyendole el nombre "personas" y la variable personas

Creamos una BBDD con XAMPP, sin colocarle campo y cuyo nombre debe ser el referenciado en los atributos de nuestro Srping

vamos a la carpeta templates (src/main/resources - templates) y creamos un archivo .html (clic-der / new / other / - / web / HTML File) y lo llamamos index.html OJO: el nombre de ese archivo es referenciado en el controlador

Dentro creamos una tabla con las etiquetas correspondientes, con los campos: - ID - NOMBRE - TELEFONO - ACCIONES Le agregamos la dependencia de ThymeLeaf, en la etiqueta de html, como (html xmlns:th="http://www.thymeleaf.org") Le agregamos el cdn de bootstrap en la ultima parte del head y agregamos las clases de CSS, ademas dentro de la etiqueta (tr) colocamos el atributo (tr th:each="persona:${personas}" ) en donde se hace referencia a "personas" del controlador - (model.addAttribute("personas", personas);).
utilizando ese atributo, referenciamos los registros de la clase Persona en el modelo (td+${persona.id}+/td)
Ahora colocamos los botones en el ultimo (td), con la clase ="btn btn"

ahora podemos correr la aplicacion con la clase main (src/main/java) -com.example.demo - CrudMysqlApplication.java y si no hay error se puede ver en:
http://localhost:8080/
